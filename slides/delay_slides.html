what's a delay line?
8 samples
  • The delay line is a simple and incredibly fun sound design tool.
  • It's a queue. First in, first out. You push a new sample in one end, and pop the oldest sample out the other.
delay lines in practice
  • In practice, we don't "move all samples from left to right".
  • Instead, we use a ring buffer.
  • The buffer has a read head and a write head.
  • The read head "trails" the write head by $d$ samples, where $d \leq n$.
  • Notice that we could have multiple read heads in multiple positions... which is cool!

ring buffer — c

#define DELAY_MAX 48000   // 1 second at 48 kHz

typedef struct {
   float    buf[DELAY_MAX];
   uint32_t w;            // write index
} DelayLine;

static float delay_process(DelayLine *d, float in,
                            uint32_t delay_samples) {
   // write the newest sample
   d->buf[d->w] = in;

   // read the sample written delay_samples steps ago
   const uint32_t r =
      (d->w + DELAY_MAX - delay_samples) % DELAY_MAX;
   const float out = d->buf[r];

   d->w = (d->w + 1) % DELAY_MAX;  // advance and wrap
   return out;
}
feedforward vs feedback
  • Feedforward delays just delay the signal by some number of samples.
  • Feedback delays have a feedback path, in which the output of the delay line is fed back into the input

where's the complexity?
  • two things make delays interesting:
  • sample interpolation
  • time-varying length
fractional delay
  • $D$ has been a whole number of samples. In practice, we rarely want a whole number of samples.
  • Tuning: A delay tuned to frequency $f$ needs $D = f_s/f$; rarely an integer especially for large $f$
  • Many effects, e.g., chorus, flanger, and vibrato modulate $D$ smoothly
  • But the buffer only has samples at integer positions. We have to interpolate.

fractional delay — linear interpolation
  • Simple and effective! Just sample a line between the two nearest points.
  • With $D = i + f$ (integer part $i$, fraction $f$): $\;\hat{x}[n-D] = (1-f)\,x[n-i] + f\,x[n-i-1]$.
  • However, it's imperfect, and induces a lowpass effect.
D = 6.30
lagrange interpolation
  • Fact: $P+1$ points uniquely define a degree-$P$ polynomial through them.
  • Each known sample is a position paired with a value.
  • Put the $P+1$ samples at integer positions $0, 1, \dots, P$, with values $a_0, \dots, a_P$. For each sample position $m$, define the basis polynomial $\ell_m$: $$\ell_m(t) \;=\; \prod_{\substack{j = 0 \\ j \neq m}}^{P} \frac{t - j}{m - j} \qquad \text{so at every integer position } k: \qquad \ell_m(k) = \begin{cases} 1 & k = m \\ 0 & k \neq m \end{cases}$$
  • $\ell_m$ is 1 at its own position $m$ and 0 at all the other positions. So scaling each $\ell_m$ by its value $a_m$ and summing gives a polynomial that hits every sample exactly: $$p(t) \;=\; \sum_{m=0}^{P} a_m\,\ell_m(t)$$
  • Evaluate $p$ at the read position $t$ to get the interpolated value.
  • Linear interpolation is Lagrange interpolation for $P{=}1$.
lagrange interpolation
  • We want $x[D]$ where $D = i + f$ (where $i\in\mathbb{Z}$ and $0 \leq f < 1$)
  • For cubic interpolation, we select the samples to the left and right of $i + f$. Those are: $x[i - 1]$, $x[i]$, $x[i + 1]$, $x[i+ 2]$
  • Assign each sample positions $-1, 0, 1, 2$, then, the read position is literally $f$.
  • Now we need to find $\ell_{-1}, \ell_0, \ell_1, \ell_2$.

lagrange interpolation — solving the first weight
  • Recall the basis polynomial $\ell_m$: $$\ell_m(t) \;=\; \prod_{\substack{j = 0 \\ j \neq m}}^{P} \frac{t - j}{m - j}$$
  • Write the definition for the sample at position $m = -1$. Here, the scary product symbol is over $j = 0, 1, 2$: $$\ell_{-1}(t) \;=\; \prod_{j=0}^{2}\frac{t-j}{-1-j} \;=\; \frac{t-0}{-1-0}\cdot\frac{t-1}{-1-1}\cdot\frac{t-2}{-1-2}$$
  • Crunch this to get: $$\ell_{-1}(t) \;=\; -\frac{t\,(t-1)\,(t-2)}{6}$$
  • Do the same for $m = 0, 1, 2$, and the interpolated sample is the weighted sum recalled from before: $$p(t) \;=\; \sum_{m} a_m\,\ell_m(t)$$ where $a_{m} = x[i + m]$
lagrange interpolation
  • We repeat the process for $m=0,1,2$.
  • $$\ell_{-1}(f) = -\frac{f(f-1)(f-2)}{6} \qquad \ell_0(f) = \frac{(f+1)(f-1)(f-2)}{2} \qquad \ell_1(f) = -\frac{(f+1)f(f-2)}{2} \qquad \ell_2(f) = \frac{(f+1)f(f-1)}{6}$$
  • The interpolated sample is : $$x[D] = x[i + F] \;\approx\; \ell_{-1}(f)\,x[i-1] + \ell_0(f)\,x[i] + \ell_1(f)\,x[i+1] + \ell_2(f)\,x[i+2]$$
  • We showed third order Lagrange, but you can do any order you want.
  • Lagrange interpolation is more accurate than linear (first order) interpolation, but it's more expensive.

fractional delay
  • Lagrange interpolation is actually a subtle low-pass filter. An allpass filter has unity gain at every frequency.
  • The Thiran allpass interplator has a maximally flat group delay, and preserves magnitude: $$H(z) = \frac{\eta + z^{-1}}{1 + \eta\,z^{-1}}, \qquad \eta = \frac{1-f}{1+f}$$
  • Unlike Lagrange it has internal state. This can cause trouble with delay length modulation.
sinc interpolation
  • Sinc interpolation is the "proper" way to reconstruct a digital signal.
  • By definition: $$x(t) \;=\; \sum_{k=-\infty}^{+\infty} x[k]\,\operatorname{sinc}(t - k), \qquad \operatorname{sinc}(t) = \frac{\sin \pi t}{\pi t}$$
  • Observe that the sinc function equals 1 at zero and is zero for all other integer values.
  • We just replace every sample with a shifted and scaled copy of the sinc function.
  • Can you see why this is impossible to implement perfectly in practice?
applications — chorus
  • A chorus makes one instrument sound like... well... a chorus
  • Recipe: copy the signal, delay the copy by roughly 10–30 ms, and mix it with the original.
  • Then, modulate $D$ slowly with a slow sinusoidal LFO.

applications — flanger
  • A flanger is a chorus with a smaller sweep value. sweep $D$ between roughly 0.1 and 10 ms, and mix with the original.
  • At delays this short you don't hear an echo, instead we get a comb filter effect.
  • A feedback delay is a comb filter. Frequencies that are harmonically related to the comb filter's tuning undergo interference.

applications — karplus-strong
  • If we tune a feedback delay to some desired frequency and put a lowpass filter in the loop, we get a string model!
  • The loop repeats every $D$ samples, so its fundamental is $f_0 = f_s/D$.
  • To emulate a pluck, we excite the model with an impulse or noise burst.
  • This is the original Karplus–Strong algorithm (1983). It has since been extended to model complex stringed instruments from around the world.
